# setTimeout 实现 setInterval

- 手写题
  setInterval 固定时间间隔
  不管任务有没有执行完 时间一到就继续触发
  setTimeout 一定时间后 执行功能
  每次执行完，再决定执行下一次任务
  Event Loop 中 setTimeout不准时，还有利用价值

## 用递归setTimeout模拟循环调度

- event loop
  为什么不用setInterval
  因为setInterval不管任务是否执行完，时间一到就触发
  可能导致任务堆积/执行重叠
  setTimeout 任务执行完，才会在下一个event loop里推入任务，更可控，更安全

## 宏任务

宏任务是宿主发起的异步任务(定时器、事件、网络请求)
微任务是引擎引起的异步任务(如 process.nextTick() Promise.resolve，observer....),事件循环每轮执行一个宏任务，然后立即清空所有的微任务(优先级更高)

JavaScript 引擎(V8) 是单线程的，它要干一堆的事情(js执行，渲染，scroll，事件响应)，干不了太耗时的事情(网络请求，定时器...)

宿主环境(浏览器或node.js)，站了出来，宿主环境是多线程的(多进程的)，有专门的线程去处理这些杂活(定时器，网络)

JS把活甩给宿主，宿主在后台(其他线程)搞定，搞定后通知JS。

setTimeout 属于浏览器的定时触发器线程。该线程独立于JS引擎，负责计时，时间到达后，将回调函数加入宏任务队列，等待主线程空闲时执行
setTimeout 设定的时间只是一个最短的等待时间，而非精确时间。实际执行时间会晚于预期

等当前的同步代码以及微任务队列清空后，才能被执行
浏览器对定时器的最小延迟有规定，最小为4ms

为了节省电量和系统资源(移动端)，当用户切换到其他标签时，降低执行频率1000毫秒以上

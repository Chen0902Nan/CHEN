# React 通用弹窗组件Modal

## 组件设计思想

函数组件
返回声明式的UI
将一段html、css、js封装成一个可以复用的组件
页面由组件构成、像搭积木一样

## 面试官心态

除了CSS和单例模式，面试官想看到我们对React Portal(传送门)、组件通信、props、children、全局状态管理、动画过渡和无障碍访问(点击页面遮罩层关闭)

React API、组件设计能力、JS设计模式、CSS、用户体验的细腻度

### 可控显示隐藏

受控组件，通过父组件传入visible属性控制弹窗显示、隐藏，同时暴露onClose回调，让父组件完全掌控弹窗状态，符合React的单向数据流理念。
保证组件状态统一，父子组件做联动逻辑(弹窗关闭后重置表单，请求接口....)

### 遮罩层（css+用户体验）

fixed定位 + z-index + opacity避免被其他元素遮挡，半透明背景，点击我们的遮罩层关闭

### ESC 键关闭
监听keydown事件，响应 esc键关闭
移除的时候，卸载事件

### 插槽内容(复用性)
React.children + title、footer props 结合的方式，支持传入标题、底部按钮区域、不同的业务场景(登录表单、提示弹窗) 提升了复用性，不用为不同的弹窗单独写组件。

## 优化

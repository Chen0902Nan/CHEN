## react中遍历数组，key有什么作用

- 开门见山
  key的本质作用：帮react 在diff过程中识别节点身份，提高复用能力，避免错误更新

React 在更新UI时，不是整个DOM全部重新渲染的，而是做一个新旧虚拟DOM的对比(diff)，然后只更新变化的部分。

如果是一个数组，key 就可以找到相应的新旧节点
有key，复用节点，(创建节点开销太大了，移动节点开销小)
没有key，可能会误删误建

key要唯一的 最好不要用index index在移动位置的时候是动态的

## diff 算法原则

- 前期约定
  两棵树做 diff，复杂度是 O(n^3) 的，因为每个节点都要去和另一棵树的全部节点对比一次，这就是 n 了，如果找到有变化的节点，执行插入、删除、修改也是 n 的复杂度。所有的节点都是这样，再乘以 n，所以是 O(n^3) 的复杂度。
  同层比对 type变了就不再比对子节点，直接销毁重建

- 在同一层里如果是列表结构，React会根据key来做匹配
  key相同，认为是同一个节点，尝试复用
  key 如果不同的话，认为是新节点，删除旧的，创建新的

- 简单diff 算法
  用移动代替重建
  旧节点 ABCD -> 新节点DCAB
  旧节点数组 [A,B,C,D]
  新节点数组 [D,C,A,B]
- 根据key，找到节点，顺序有没有需要调整的地方？
  关键规则：按新数组顺序遍历，去旧数组找key相同的节点，用lastIndex去判断是否需要移动
  lastIndex 在当前遍历新数组的过程中，所有已复用的节点中，在旧数组中最大的那个下标
  它的设计目标：通过判断一个可复用节点在旧数组中的位置是否落后于之前所有已处理节点的最远位置，来决定是否需要移动

执行过程：

1. 初始化lastIndex=0
2. 遍历新数组第一个节点D
   去旧数组中key D，旧下标3
3. 如果大于lastIndex(0) 不用移动
   lastIndex=3
4. 遍历新数组第二个节点C
   在旧数组中找C发现旧下标2
   2<lastIndex(3) 需要移动
   移动到新数组第二位 D的后面
5. 遍历A
   旧下标 0
   0<lastIndex(3) 需要移动
   移动到新数组第三位 C的后面
6. B
   旧下标 1
   1<lastIndex(3) 需要移动
   移动到新数组的第四位 A后面

7. 清理节点
   删除，新增

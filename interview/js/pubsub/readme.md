## 发布订阅者模式

常用的面向对象设计模式。定义了一种一对多的依赖关系，让多个订阅者通过事件中心(listeners)监听发布者的消息，实现对象之间的解耦通信。

抖音|哔哩哔哩 平台，存着订阅者们和发布者的事件中心，不需要发布者一个一个的和订阅者交流

## 观察者模式

对象间一对多的依赖关系，目标状态发生改变时自动通知并更新所有观察者，和订阅发布者模式很像
两者都基于事件机制，旨在实现对象间的一对多通信。当一方状态改变时，依赖方会自动收到通知并更新，从而实现轻松解耦。

- 两者的区别
  发布订阅者通过事件中心转发消息，发布者不直接联系订阅者，观察者是对象直接订阅目标，变化时被直接通知
  发布订阅模式：像用微信订阅公众号，公众号发消息到"平台"，平台推送给你，
  公众号根本不知道你是谁
  观察者模式 像你关注周杰伦，他一发动态，直接通知你，是你和他的直接关系
  发布订阅中间多了一层 中介(事件中心)，观察者是直接盯着目标

- 应用场景
  发布订阅
  DOM 事件机制
  document.addEventListenner('click',fn)
  浏览器负责分发，元素不知道谁在监听

  组件只关心订阅状态,不关心谁改的
  Event Bus mitt
  自定义事件 EventEmitter 自己实现的事件中心
  node eventEmitter是核心

# 打家劫舍

动态规划 dp 在不同数据结构上(线性数组、环形、树)的应用

- 最值问题 dp 解决
- dp 哪些特征
  递归 自顶向下 相似的问题 树形结构
  退出条件 程序栈
  爆栈 重复计算
- 记录本 取出重复计算
- dp 自底向上
  - 重叠子结构
    大问题会反复用到很多相同的小子问题结果，重复计算浪费性能，需要缓存
  - 最优子结构
    最值问题，每一步都是最值
    dp[i]
    dp[i][j]
    大问题的最优解，可以由若干个子问题的最优解导出来
  - 状态转移方程
    自底向上
    迭代状态 变量是有状态

## 爬楼梯看待dp

1. 推结论
   dp[i]=爬到i阶的总方法数
2. 状态转移方程
   i阶
   i-1 走一步
   i-2 走两步 重叠子结构
   i-1 -> i-2 i-3
   最优结构
   dp[i]=dp[i-1]+dp[i-2]
3. 初始边界
   dp[1]=1
   dp[2]=2

## dp 偷盗一

dp[i] 偷|不偷
偷：dp[i-1]就不要 dp[i-2] + nums[i]
不偷：d[i]

do[i]=Math.max(dp[i-1],dp[i-2]+nums[i])

## dp 偷盗二 环形版

- 第一个房子和最后一个房子不能同时偷
- 分两种情况 nums[0]-nums[n-2] | nums[1]-nums[n-1]
- 最终取两者更大的一种

## 树形结构

直线版：数组，线性相邻不能偷
环形版：首尾相连，多一个约束
树状：父子两点直接相连，不能同时偷
偷了父节点，两个孩子不能偷
rob=left.notRob + right.notRob + cur.val
不偷父节点 两个孩子可以偷或不偷 取最大
notRob=max(left.rob,left.notRob) + max(right.rob,right.notRob)

# k个一组翻转链表

## 思路

- 封装一个翻转函数
- k个一组
  - 先数够k个节点，把这节链表摘下来反转
  - 反转后把这段接回主链表
- dummy节点
  用来简化边界处理
  1->2->3->4->5 k=2
  dummy ->1
  pre 前序节点 永远指向当前待处理分组的第一个节点
  他负责在反转后 把新头接上
  end 末尾节点 侦查兵的角色 负责向前走 k步，帮我们找到
  当前分组的最后一个节点
  end
  pre
  dummy ->1 ->2 ->3 ->4 ->5

  记下关键位置，准备“摘除”
  现在我们已经圈定了范围，但直接动手会弄丢后面的链表，所以得先记下两个关键位置： start (起始节点)：pre 的下一个节点，也就是当前组的第一个节点 1。反转后，它会变成这一组的尾巴。 nextGroup (下一组起点)：end 的下一个节点，也就是 3。这是我们反转时不能弄丢的“后半段”链表。

  第四步：执行“摘除”操作 最关键的一步来了！为了能把 [1 -> 2] 这段独立出来反转，我们需要暂时切断它和后面链表的联系。 操作就是：end.next = null 执行后，链表变成了两部分： 待处理部分：dummy -> 1 -> 2 -> null 剩余部分：3 -> 4 -> 5 (由 nextGroup 指针牢牢抓住) 到这一步，“数够k 个、摘下来”的操作就完美完成了。接下来就可以放心地对 [1 -> 2] 这个小链表进行反转，而不用担心整个链表会断开丢失。

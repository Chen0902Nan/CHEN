# 移动零

- 空间复杂度
  不能额外增加O(n)的空间
- 同时保持非零元素的相对顺序
  稳定性

## 稳定性

在操作后仍然保持元素的相对顺序不变

### 插入排序

摸扑克牌

将数组分为已排序(手上)和未排好的(没有摸的牌，待排序)两部分，每次从未排序部分取出一个元素(摸牌)，将其插入到已排序的正确位置，直到所有元素都完成插入，时间复杂度O(n^2)

不稳定
[5,8,5,2]
有两个相同的值，[5a,8,5b,2]
值相同，顺序不同,[2,5b,5a,8]

### 选择排序

1. 第一轮[5a,8,5b,2]找到最小值，是2，将2与第一个位置元素的5a进行交换
   5a,就到了5b的后面，稳定性就破坏了
2. 第二轮 在剩余未排序的部分 [8,5b,5a] 找到最小值，5b，和8交换
   [2,5b,8,5a]
3. 第三轮 在剩余未排序[8,5a,]中找到最小值，5a 和8交换
   [2,5b,5a,8]

### 原地操作 + O(1)空间 + 稳定性

用快慢指针，从0开始，快指针指向元素非零的时候就交换快慢指针元素，同时慢指针+1。
第一次遍历完后就把非零元素都挪到了前面，再遍历一遍慢指针，填充0

# 正则匹配邮箱

正则就是用一套特定的符号规则，来描述和匹配一类字符串的通用模式
qq邮箱？
用户的输入是不是一个合格的邮箱？
754211506@qq.com
邮箱的基本结构
xxx@xx.xxx

- 基本雏形

```js
const reg = /.+@.+\..+/;
let str = "754211506@qq.com";
console.log(reg.test(str));
```

# 拿硬币

最值问题，动态规划(复杂)|贪心(简单)

贪心就是每一步都选当前最优，比如找零钱，选择最大面额，图省事但不一定全局最优。

动态规划会记录子问题最优解，综合得到全局最优，更稳定，但更复杂

# Event Loop

- 渲染进程的主线程有太多工作要做
  消息队列 + Event Loop(JS 执行机制)

- 一次工作中
  - script 是一个宏任务
  - v8引擎JS 单线程 同步代码 快速执行完 调用栈
    执行时间不能太长,优先级高的任务
  - 耗时性任务等 异步的，不能被阻塞
  - 微任务队列 (promise,mutation) 时机，resolve 后进入
  - 非队列(页面的渲染 -> 重排重绘，动画) -> 掉帧，页面的卡顿 垃圾回收(JS自己不定期进行垃圾回收)
  - 宏任务队列 (JS执行机制开始于宏任务,script就是)

  JS执行 是从 任务开始的 第一个宏任务是script

- 宏任务
  定时器

JS 在渲染进程的主线程上运行，是单线程的。
执行模型依赖：宏任务队列 + 微任务队列 + Event Loop。
页面加载的第一个宏任务是解析并执行 <script>。
每一轮 Event Loop 大致流程：
执行一个宏任务（包括其同步代码）
清空当前所有微任务（可能多次产生新微任务，继续清）
尝试执行渲染更新（如果这一帧还有剩余时间）
进入下一轮，取下一个宏任务

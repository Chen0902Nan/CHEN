JavaScript
function render(element, container) {
  const dom =
    element.type === 'TEXT_ELEMENT'
      ? document.createTextNode(element.props.nodeValue)  // 直接给值，别空字符串
      : document.createElement(element.type)

  const isProperty = key => key !== 'children'

  Object.keys(element.props)
    .filter(isProperty)
    .forEach(name => {
      if (name === 'style') {
        // 字符串 style 专用处理
        dom.style.cssText = element.props.style
        // 或者 dom.setAttribute('style', element.props.style)
      } else {
        dom[name] = element.props[name]
      }
    })

  element.props.children.forEach(child => render(child, dom))
  container.appendChild(dom)
}

这样你的 salmon 背景和 text-align:right 就都能出来了。

什么是 Babel？

Babel 就是一个 JavaScript“翻译机 / 编译器”。

它最出名的两个工作：

  1. 把新语法（ES2023、ES2024）翻译成老浏览器能跑的 ES5
  2. 把 JSX 这种“看起来像 HTML 但其实是 JS”的语法，翻译成普通的 JavaScript 函数调用

你写的 JSX：

jsx
<div style="background:salmon">
  <h1>Hello world</h1>
</div>

被 Babel（通过 @babel/preset-react）翻译后变成：

JavaScript
Didact.createElement(
  "div",
  { style: "background:salmon" },
  Didact.createElement("h1", null, "Hello world")
)

所以没有 Babel，你直接在浏览器里写 JSX 是会语法错误的（浏览器根本不认识 <div> 这种标签写在 JS 里）。

你项目里用的是最原始的方式：

  • 写 /** @jsx Didact.createElement */ 告诉 Babel：请把 JSX 转成 Didact.createElement 调用，而不是默认的 React.createElement

什么是 Didact？

Didact = 你自己手写的、极简版的 React

它是很多“Build your own React” 教程里最经典的命名（来自 2017–2018 年那篇非常火的文章系列）。

为什么叫 Didact？

  • 英文里 “didactic” = 说教的、教导的
  • 作者想表达：这个库就是用来教你 React 底层原理的，不是真的拿来写业务

所以你现在看到的这个项目，本质上就是：

“React 的教学版山寨货”

它目前只实现了 React 非常早期、最核心的两部分：

  • createElement → 产生虚拟 DOM
  • render → 一次性把虚拟 DOM 渲染成真实 DOM（没有更新、没有 diff、没有 Fiber）

后续阶段（大多数教程会继续写）：

  • reconcile / commit 阶段
  • setState + 批量更新
  • 函数组件 + Hooks 雏形
  • Fiber（可中断渲染）

总结一句话给你背下来

  • Babel：把 JSX 翻译成 createElement 调用
  • Didact：你自己写的假 React，用来学习真 React 原理
  • 当前项目：用 Babel + Didact 实现了一个能把 JSX 渲染到页面的极简框架（第一、二阶段）

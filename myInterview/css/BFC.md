# BFC

- Block Fomating Context 块级格式化上下文，是CSS中独立的(不受外界影响)的布局区域
  对于两列式、三列式的布局起到了关键性的支持

- 核心规则
  [创建规则]、[布局规则]、[交互规则]

- 主要解决的问题

1. 外边距重叠：父子元素或兄弟元素之间的margin莫名其妙地被合并了，导致间距不对
2. 高度塌陷：父元素里的子元素浮动了，父元素因脱离文档流撑不开高度
3. 布局错乱：两个元素相邻，一个浮动，另一个文字被挤到浮动元素下方，导致布局混乱

- 创建规则
  - html就是最大的BFC元素，块级元素不一定是BFC元素，得创建
  - display:flow-root 最推荐的方式，没有副作用
  - overflow:hidden 最常见的方式，但时要注意意外截断和响应式灾难
  - float:left 只要float属性不为none，就会让元素脱离文档流，一般不建议
  - position:absolute或fixed 同样会脱离文档流
  - display:inline-block等 也能触发，但是会改变元素本身的显示模式

- 布局规则
  垂直排列
  - BFC 内部块级盒子从上到下依次垂直排列
  - 边距的折叠 相邻块级盒子的垂直margin会折叠，取最大值，非相加
  - 触边对齐
    每个盒子左外边缘紧贴BFC容器的左边缘，即使有浮动元素在
  - 独立计算
    BFC是一个独立渲染的区域，不受外界影响

- BFC与外部交互的规则
  - 包含内部浮动，BFC会包裹所有内部浮动元素(解决父元素高度塌陷问题)
  - 排除外部浮动，BFC区域不与外部浮动元素重叠

# 前端常用布局方式

- 传统流式布局 Block + Inline-block
  这是最基础的布局，它是HTML文档流的默认行为
  - 核心逻辑：display:block 元素独占一行，inline-block 元素并排排列
  - 适用场景：简单的图文混排、按钮组、文字内容的布局
  - 局限：现代开发中要减少使用inline-block布局，因为它存在"换行符"间隙的问题,需要通过设置父级 font-size:0来解决

- 浮动布局 Float
  过去十年的主力军
  - 核心逻辑：通过float:left/right 来让元素脱离文档流
  - 痛点：父元素高度塌陷(必须清除浮动BFC)，且不好控制对齐
  - 基本已被淘汰，除了少数实现文字环绕图片的需求

- Flexbox 弹性布局 一维布局的王者
  这是目前前端开发中最常用、性价比最好的布局方式
  - 核心：通过display:flex 将容器变为弹性盒子，可以轻松实现子元素的对齐、空间分布和自动缩放
  - 常用属性：
    justify-content主轴对齐(默认为水平方向)
    align-items: 交叉轴对齐(默认为垂直方向)
    flex-grow/shrink:子元素伸缩比例控制
    flex:是否压缩，以及比例
  - 适用场景：几乎所有

- Grid 网格布局 二维布局王者
  Grid处理的是行和列同时变化的复杂布局
  - 核心逻辑：将容器划分为网格系统。可以通过grid-template-columns和grid-template-rows定义复杂的棚格
  - 必杀技：fr单位(剩余空间份数)和grid-area(命名布局)。它能实现很多以前需要嵌套大量div才能完成的复杂排版
  - 适合场景：Dashboard仪表盘、复杂的数据图表页、画廊式布局、复杂的响应式页面架构

- 定位布局 Position
  核心逻辑：position:absolute/fixed/relative/sticky
  关键点：
  absolute 脱离文档流 基于最近的非static父级定位
  fixed 相对于视口定位 常用于弹窗 置顶导航
  sticky 实现吸顶效果 不需要复杂的js监听滚动
  - 适用场景：图层遮罩、按钮悬浮、复杂的定位标注

## 总结

1. 大框架布局 首选Grid 能够帮助我们确定整个页面骨架 比如 Header、Sidebar、Main Content、Footer区域划分
2. 组件级布局 选用Flex 比如按钮内部的图标对齐 导航栏内部的菜单排布 卡片的布局

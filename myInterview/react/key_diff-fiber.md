# react中key的作用

react中的key是一个特殊的字符串属性，是react协调算法diff过程中的关键，它保证diff算法高效识别元素变化，保持组件状态稳定。它必须是唯一值，不能用index来作为react的key

# dfff算法

react采用diff算法对虚拟DOM进行对比，更新DOM，渲染界面。常规的diff算法的时间复杂度是O(n^3)，react的diff算法采用三个规则，把diff算法的时间复杂度降到了O(n)。

- 同级比较
  在 Web 开发中，DOM 节点跨层级移动的情况非常少见，所以 React 在进行 Diff 时，会按照树的层级结构，一层一层地向下遍历。它只会对比同一个父节点下的子节点列表。如果一个节点在旧树和新树中的层级发生了变化（比如从爷爷节点下跑到了孙子节点下），React 不会尝试去移动它，而是直接判定旧节点被删除，新节点被创建。
- 同类型比较
  React 首先检查新旧节点的 type 属性（即标签名或组件类）。
  类型不同：如果一个是 <div>，另一个是 <span>，React 会认为它们完全不同。它会直接销毁旧节点及其所有子节点，然后挂载新节点。
  类型相同：如果都是 <div> 或都是同一个组件，React 会认为这两个节点是同一个东西（可以复用）。它会保留该 DOM 节点，仅对比并更新发生变化的属性（如 className, style），然后继续递归对比子节点。
- key相同
  当 React 对比同一层级的多个子节点（列表）时，如果仅仅依靠“位置索引”来判断，一旦列表顺序改变，就会导致大量误判。引入 key 后，React 会优先根据 key 来匹配新旧节点。
  Key 相同：React 认为这是同一个节点，只是位置可能变了。它会复用该节点（保持状态），并执行移动操作（如果需要）。
  Key 不同/不存在：React 会认为这是一个新节点（需要创建）或者旧节点已被删除。

- lastPlaceIndex
  当在进行同一层级的多个子节点的diff比较的时候，diff有两轮循环，第一轮是简单的比较，如果新旧节点的类型相同,key也相同，react认为这个节点没有变化，可以直接复用，同时更新lastIndex。lastIndex表示在当前新列表中，最后一个不需要移动的节点在旧列表的位置，初始的时候为0，如果新节点的在旧列表的index小于lastIndex,就说明当前节点在旧节点中位置更靠前，现在跑到了靠后的位置，顺序错乱，需要移动。如果新节点在旧列表的index>=lastIndex，说明新节点在旧列表的位置靠后，相对位置没变，不需要移动，此时更新lastIndex。

  当key不同的时候，react会直接跳出第一轮循环，将旧列表的剩余节点全部塞进一个Map中{key:key,value:fiber},将新列表中的剩余节点到Map去进行查找对比，如果找到key相同的就，取出来，通过 lastPlacedIndex 判断它是不是挪动了位置，然后复用。如果没有，说明是新节点，原地创建一个。对比完后，会销毁旧列表中存在，但新列表中没有的节点。

# Fiber

type：组件的类型（如 div, MyComponent）。
props：组件的属性。
child / sibling / return：指向子节点、兄弟节点和父节点的指针（这是实现链表遍历的关键）。
stateNode：指向真实的 DOM 节点或组件实例。
expirationTime / lane：任务的优先级，用来决定哪个任务先执行。

是在React16之后引入的一个全新的架构，他把原来虚拟DOM树切割成了一个个小的fiber,从树状结构变成了链表，每个fiber都有三个指针，分别指向父节点，子节点，右边最近的兄弟节点。它使虚拟DOM的更新变得可暂停，同时可以分批次处理，实现优先级调度，以及Suspense,useTransition等现代React特性。

Render 阶段：异步且可中断。React 在此阶段执行 Diff，对比currentTree和JSX，来构建workPlaceIndex,根据优先级调度任务。如果用户有输入等高优先级操作，React 会放弃当前的 WIP 树构建，优先响应用户。
Commit 阶段：同步且不可中断。将 Diff 计算出的 Deletion、Placement、Update 等标记一次性映射到真实 DOM 上，渲染出真正的DOM，确保 UI 渲染的完整性。”

# 总结

“总结来说，React 的这套体系并不是单纯追求计算速度的最快，而是在追求‘响应速度’。它通过 Fiber 架构夺回了主线程的控制权，配合 Diff 算法的复用机制，解决了复杂交互下的掉帧问题，这也是 React 能够支撑超大型前端项目的底层基石。”

React 19 Compiler 的介入：
它在编译阶段（Babel 插件层）通过静态分析，自动为你的代码注入了类似 useMemo 的逻辑。它会识别哪些变量依赖于哪些原始数据，并自动生成缓存。
对 Diff 的影响：如果一个组件的 Props 和 State 经过 Compiler 处理后被判定为“未改变”，React 会在 Render 阶段直接跳过该组件的 reconcileChildren 过程。这意味着 Diff 算法压根就不会启动。

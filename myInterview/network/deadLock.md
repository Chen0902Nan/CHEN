# 死锁

- 死锁其实就是在多个进程(或线程)在执行过程中，因抢夺资源而造成的一种相互等待，如果没有外力介入，它们都将无法继续执行任务

比喻：就想是两个人过独木桥，A想先去到B的方向，B想先去到A的方向，但是两个人谁也不让谁，都在独木桥上耗着

- 发生死锁的四个条件，缺一不可

1. 互斥条件：资源是独占的，某一种资源被别的线程需要，但是只有A线程有
2. 请求与保持条件：线程已经持有了至少一种资源，但又去请求新的资源，而那个资源正被别的线程持有。它在等待新资源的同时，又不肯释放自己手里的资源
3. 不可剥夺：线程获取到的资源，在自己使用完之前，不能被别人强行夺走
4. 循环等待条件：存在一个线程链，线程A在等待B的资源，B在等待A的资源，C在等待A的资源，三者形成了一个循环等待的圈

- 如何解决死锁
  其实就是破坏形成死锁的条件，破坏其中任意一个，都不再存在死锁

1. 打破循环等待：规定线程按照一定顺序获取资源，比如A要先获得资源1再获得资源2，B要先获得资源2，再获得资源1，那就指定规则，都先获得资源1，再2
2. 打破请求保持：在开始执行前，就把所有资源全都锁上，如果有其中一个锁不到，就一个也不要，放弃已持有的资源，等待下一次尝试
3. 打破不可剥夺：给锁设置一个等待时间，如果超时了也拿不到资源，就主动释放已持有的资源，过一会再进行尝试
4. 银行家算法：就是在分配资源前先进行安全检查，确保系统不会进入不安全状态。不过实际高并发系统中，算法开销大，很少使用

## 25——k个一组翻转链表

这题难度其实不大，主要是理清楚链表之间的节点，构造一个辅助反转函数，每次都移动k次cur指针，如果移动期间cur不存在了，证明剩下的链表不需要翻转，可以直接返回。注意的是辅助函数的返回值，以及一开始的start翻转后其实已经到了末尾，要重新桥接。

## 160——相交链表

这题有两种做法

- 第一种就是先对齐起点，再遍历链表，找到相同就返回，没找到相同就在最后返回null
- 第二种采用 (x+z)+y=(y+z)+x的巧妙思维

## 234——回文链表

利用快慢指针，先找到链表的中间节点，反转后半部分的链表，再进行两部分链表的对比，判断是否相同即可

## 141——环形链表

利用双指针进行判断环形链表

## 142——环形链表2

其实也是利用双指针，但是一个巧妙的数学思维，当判断有环后，其实再走a步就到了入环口，a就是从head到入环口的距离。所以判断有环后，让一个left指针从头开始，一个slow指针从原地开始，两个指针同时运动，每次一步，相遇的时候返回left即可

## 2——两数相加

这题要考虑的就是进位的处理

- 解法一 迭代
  利用carry=0处理进位，每次都更新并进行传递，carry=Math.floor(sum/10)就是下一次的进位，val=sum%10
- 解法二 递归
  退出条件就是l1、l2、carry都为0了。同样进行更新，递归 return new ListNode(sum%10,addTwoNumbers(l1,l2,carry)),注意要先更新carry
